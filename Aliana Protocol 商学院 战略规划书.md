## 一、背景与战略定位

### 1.1 为什么需要一所商学院？

阿里协议的核心信条是 **“数学是唯一的领袖，代码即法律，时间是最好的杠杆”**。然而，再精密的合约、再透明的机制，如果参与者无法理解其运行逻辑，就无法建立真正的信任。

**行业困境**：

- Web3 用户平均空投完成率仅 **8-12%**
- 7 天留存率只有 **5-10%**
- 超过 90% 的潜在用户在第一次接触后就流失了
- 根本原因：**“看不懂，所以不敢动”**

商学院正是为了解决这个认知鸿沟而设立——它不是简单的“用户教育”，而是一套系统化的 **认知基建工程**。

### 1.2 商学院的核心定位

> **“认知即共识，教育即治理。”**

Aliana 商学院不是传统意义上的培训机构，而是协议生态的 **“认知基础设施”**：
- 将复杂机制翻译为可理解语言
- 将用户转化为共建者
- 将共识固化为社区文化

### 1.3 商学院的差异化优势

| 维度 | 优势 |
|------|------|
| **机制即教材** | 协议的预付利息、奖励额度、FIFO 撮合本身就是最生动的案例教学素材 |
| **社区即生源** | 已有超过 2 万名活跃会员构成天然的生源基础 |
| **实践即检验** | 学员可以直接在真实协议环境中验证所学，形成“学-练-用”闭环 |
| **治理即毕业** | 参与链上治理投票本身就是商学院最核心的“毕业设计” |


## 二、使命、愿景与价值观

### 2.1 使命

**让每一位参与者理解代码背后的数学逻辑，让信任建立在可验证的认知之上。**

### 2.2 愿景

成为 DeFi 领域最具影响力的去中心化教育平台，培养十万名能够独立理解、验证和参与链上协议的 **“数学公民”**。

> **行业验证**：Binance Academy 在 2024 年触达了 4,400 万人；Binance “Blockchain for Vietnam” 计划两年投资超过 100 万美元。教育基建不是可选项，而是协议长期生存的必选项。

### 2.3 核心价值观

- **数学为本**：一切课程内容以合约逻辑和数学原理为根基
- **链上验证**：每一堂课都能在链上找到对应的验证场景
- **共建共享**：课程内容由社区贡献、社区审核、社区迭代
- **长期主义**：不追求短期流量，专注于培养深度理解者和长期共建者


## 三、核心逻辑：商学院如何与主协议协同

商学院与协议形成 **“教育-实践-治理”** 的闭环：

```
商学院（认知建设）→ 学员深度理解协议 → 参与存款/P2P/节点/治理
                    ↑                              ↓
          治理反馈优化课程 ← 实践数据反哺教学 ← 产生真实链上行为
```

**协同关系**：

| 角色 | 核心任务 | 与商学院的协同 |
|------|---------|---------------|
| **主协议** | 提供金融基础设施 | 商学院为协议输送深度用户，降低流失率 |
| **社区工作室** | 线下落地与社区扩张 | 商学院为工作室培训合格的建设者 |
| **商学院** | 认知建设与人才输出 | 连接协议与社区的“认知桥梁” |

## 五、组织架构（深化版）

商学院采用 **“DAO 驱动的三层治理架构”** ，在去中心化决策与高效执行之间寻求平衡。

### 5.1 三层治理架构总览

```
┌─────────────────────────────────────────────────────────────────┐
│                   第一层：最高决策层（DAO 治理）                   │
│  全体社区成员（ALP 持有者/学员/讲师）通过链上投票决定重大事项        │
└─────────────────────────────────────────────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────────┐
│                   第二层：执行管理层（教务委员会）                  │
│          5-7 名成员，由社区选举产生，负责日常运营与执行             │
└─────────────────────────────────────────────────────────────────┘
                                  │
                    ┌─────────────┼─────────────┐
                    ▼             ▼             ▼
┌─────────────────────────────────────────────────────────────────┐
│                   第三层：社区参与层（角色体系）                    │
│    认证讲师  │  社区导师  │  课程贡献者  │  学员代表  │  校友会    │
└─────────────────────────────────────────────────────────────────┘
                                  │
                                  ▼
┌─────────────────────────────────────────────────────────────────┐
│                        顾问委员会（咨询机构）                      │
│        由协议核心贡献者、外部专家组成，提供战略建议与专业支持         │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 第一层：最高决策层（DAO 治理）

**治理主体**：

- **ALP 持有者**：持有任意数量 ALP 即可参与投票
- **商学院认证持有者**：完成基础/进阶/专家认证的学员
- **讲师与导师**：通过商学院认证的讲师和社区导师

**投票机制**：采用 **“一人一票 + 权重系数”** 混合模式。持有商学院毕业 NFT 或认证讲师 NFT 的地址可获得额外权重加成（1.5 倍）。

**决策范围**：

| 决策事项 | 通过门槛 |
|---------|---------|
| 章程修改 | ≥66% 赞成票 |
| 教务委员会选举/罢免 | ≥55% 赞成票 |
| 年度预算审批 | ≥55% 赞成票 |
| 战略方向调整 | ≥60% 赞成票 |
| 紧急暂停 | ≥50% 赞成票（24小时快速投票） |

**投票参数**：
- 提案发起：需获得至少 50 个地址附议
- 投票周期：常规 5 天 / 紧急 24 小时
- 法定人数：≥100 个地址
- 执行延迟：通过后 48 小时

**治理激励**：
- 参与投票获得小额 ALP 奖励
- 提案被采纳获得额外 ALP 奖励
- 连续参与 5 次以上投票获得“治理参与徽章”NFT

### 5.3 第二层：执行管理层（教务委员会）

**委员会构成**（5-7 名）：

| 席位类型 | 数量 | 产生方式 |
|---------|------|---------|
| 社区选举席 | 3-5 席 | 社区投票选举 |
| 协议代表席 | 1 席 | 主协议治理指定 |
| 创始席 | 1 席 | 创始团队指定（6个月后转为选举席） |

**任期**：6 个月，可连选连任（最多 2 届）

**选举时间**：每年 1 月和 7 月

**候选人资格**：
- 完成商学院专家路径认证
- 在社区中有 6 个月以上活跃贡献记录
- 获得至少 30 个社区成员提名

**核心职责**：

| 职责领域 | 具体内容 |
|---------|---------|
| 课程管理 | 课程体系规划、质量审核、教材标准化 |
| 讲师管理 | 讲师认证与考核、培训、激励执行 |
| 学员管理 | 招募、进度追踪、认证发放审核 |
| 预算执行 | 编制、支出审批（<5,000 USDT）、财务报告 |
| 战略提案 | 向 DAO 提交战略提案 |
| 社区沟通 | 工作报告、反馈收集、治理讨论 |

**决策机制**：
- 日常事务：简单多数通过（≥50%+1）
- 重大事项（如课程体系重大调整）：需 ≥2/3 同意
- 决议需在 48 小时内向社区公示

**委员会激励**：

| 激励类型 | 说明 |
|---------|------|
| 基础津贴 | 每月 500-1,000 USDT |
| 绩效奖励 | 根据季度 KPI 完成情况 |
| 治理代币 | 获得 ALP 奖励 |
| 荣誉激励 | 官网公示、专属 NFT |

### 5.4 第三层：社区参与层（角色体系）

**角色总览**：

| 角色 | 核心职责 | 准入条件 | 激励方式 |
|------|---------|---------|---------|
| **认证讲师** | 课程开发与授课 | 专家认证 + 试讲考核 | 授课报酬 + 讲师 NFT |
| **社区导师** | 学员答疑、实践指导 | 进阶认证 + 讲师推荐 | 导师津贴 + 导师 NFT |
| **课程贡献者** | 课件制作、翻译 | 开放申请 + 内容审核 | 贡献者徽章 + ALP |
| **学员代表** | 反馈体验、参与治理 | 在读学员选举 | 代表 NFT + 治理优先权 |
| **校友会** | 网络建设、持续学习 | 完成任一认证 | 校友 NFT + 专属活动 |

**角色晋升路径**：

```
学员 → 认证学员（基础）→ 认证学员（进阶）→ 认证学员（专家）
    → 认证讲师 → 社区导师 / 教务委员候选人
```

### 5.5 顾问委员会

**定位**：咨询机构，无投票权，仅提供战略建议和专业支持。

**核心职能**：
- 为战略方向提供专业建议
- 为课程内容提供学术和技术审核
- 为重大决策提供第三方视角
- 帮助建立外部合作网络

**成员构成**：

| 成员类型 | 数量 | 说明 |
|---------|------|------|
| 协议核心贡献者 | 1-2 人 | 确保方向一致 |
| 外部教育专家 | 1-2 人 | 区块链/DeFi 学者 |
| 行业实践者 | 1-2 人 | 资深从业者 |
| 社区代表 | 1 人 | 社区选举产生 |

**任期**：12 个月，可续任

### 5.6 关键治理机制

**制衡机制**：

| 制衡方式 | 说明 |
|---------|------|
| 教务委员会 vs DAO | 委员会执行日常，重大决策须经 DAO 投票 |
| 选举 vs 罢免 | 委员会成员由选举产生，也可由社区罢免 |
| 任期限制 | 最多连任 2 届（共 12 个月） |

**冲突解决机制**：
1. 教务委员会内部协商
2. 提交社区公开讨论
3. 发起 DAO 投票裁决
4. 涉及主协议利益的，由主协议治理介入

**透明度要求**：
- 教务委员会每月发布工作报告
- 每季度发布财务报告
- 所有治理提案和投票结果公开存档
- 讲师认证和学员毕业数据链上可查


## 六、课程体系

### 6.1 课程设计原则

采用 **“理论 + 技术分析 + 模拟应用”** 混合模式，分三个层次递进：
- **认知层**：区块链、DeFi 和阿里协议的基础认知
- **机制层**：协议核心机制与数学逻辑
- **实践层**：真实环境操作、验证、参与

### 6.2 课程模块

**第一模块：DeFi 基础认知（必修）**

| 课程 | 课时 |
|------|------|
| 区块链与 DeFi 导论 | 2 小时 |
| 钱包与链上交互 | 2 小时 |
| 代币经济学入门 | 2 小时 |
| DeFi 风险识别 | 2 小时 |

**第二模块：阿里协议机制深度（必修）**

| 课程 | 课时 |
|------|------|
| 协议架构总览 | 2 小时 |
| 预付利息机制 | 2 小时 |
| 奖励额度与通缩飞轮 | 3 小时 |
| FIFO 订单簿与 1:1 锚定 | 2 小时 |
| ALP 代币经济学 | 2 小时 |
| 节点生态与长期主义 | 2 小时 |
| 链上治理 | 2 小时 |

**第三模块：实战演练（选修+必修）**

| 课程 | 课时 |
|------|------|
| 链上数据解读 | 2 小时 |
| P2P 做市模拟 | 2 小时 |
| 治理提案模拟 | 2 小时 |
| 节点收益计算 | 1 小时 |
| 风险场景推演 | 2 小时 |

**第四模块：进阶专题与特聘讲座（选修）**

| 课程 | 课时 | 讲师 |

### 6.3 学习路径

- **入门路径（2 周）** ：第一模块 + 第二模块（前 3 门）→ “Aliana 基础认证”
- **进阶路径（4 周）** ：入门路径 + 第二模块（剩余 4 门）+ 第三模块（前 3 门）→ “Aliana 进阶认证”
- **专家路径（8 周）** ：进阶路径 + 第三模块（剩余 2 门）+ 第四模块（至少 2 门）+ 毕业项目 → “Aliana 专家认证”

## 九、商学院如何真正赋能阿里协议

### 9.1 降低认知门槛 → 扩大有效用户基数

DeFi 最大的增长瓶颈是 **认知门槛太高**。商学院将复杂机制翻译为可理解的模块化知识。

### 9.2 学习即预演 → 将教育行为转化为链上行为
